从0开始部署hexo到github

以下均在Windows系统下操作

注意：以下均在Windows系统下操作

1.安装环境：

git：官网下载 （选择win系统下载）
nodejs：官网下载 （选择LTS版本下载）

验证：

node -v
git -v
npm -v

这里要提一下npm换镜像源的问题：

npm config get registry  # 查看当前镜像
npm config set registry https://registry.npmmirror.com # 切换镜像

没有使用淘宝源是因为听说它在去年六月下架了，所以防范未然，当然如果有代理会方便很多

2.hexo安装：

npm install hexo-cli -g

3.部署Blog：

新建一个博客文件夹，我命名为test，进入test，鼠标右键，通过git bash打开

运行下面的命令，初始化->生成静态页面->启动服务器

hexo init
hexo generate  # 简写 hexo g
hexo server  # 简写 hexo s

复制快捷键：ctrl+insert 粘贴快捷键：shift+insert
复制粘贴http://localhost:4000/到浏览器打开

按下ctrl+c停止服务，然后我们需要配置一下本地的ssh连接github，在当前git窗口下输入下列命令，一直按回车即可

cd ~
mkdir ~/.ssh
cd .ssh
git config --global user.name "你的名字"  
git config --global user.email "你的邮箱"
ssh-keygen -t rsa -C "你的email"

在.ssh文件夹下会生成id_rsa和id_rsa.pub两个文件，用记事本打开id_rsa.pub，全选复制

在github的setting里找到SSH and GPG keys，然后new SHH key，然后粘贴

最后验证一下：

ssh -T git@github.com

如果是第一次的会提示是否continue，输入yes就会看到：
You’ve successfully authenticated, but GitHub does not provide shell access 。

这就表示已成功连上github

如果你没有github账号，你需要先注册一个

然后你需要新建一个仓库：

接下来复制SSH：

然后我们打开text目录，找到并打开_config.yml文件，第104行代码，注意冒号后面有空格

deploy:
  type: git
  repo: # 这里填刚刚复制的ssh地址
  branch: main

然后在git运行（hexo clean 一定要运行一下，不然可能上传不了）

# 如果报错
ERROR Deployer not found: git
# 安装插件
npm install hexo-deployer-git --save
hexo clean # 这里不能简写哦，反正我的不行
hexo deploy # 简写 hexo d

如果加速了也无法连接github，比较玄学，重新启动git试试，github服务器经常抽风，不行就多来几次
