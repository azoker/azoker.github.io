利用JWT生成token并且校验token

Gitee地址：https://gitee.com/azoker/jwt-demo.git

1. JwtUtil类

JwtUtil.java

这个类是拿来生成 token 和解析 token 的。

创建 token 需要三个参数：密钥、过期时间、载荷（Claims）。

在 JWT 依赖中由JwtBuilder这么一个类，我们可以直接调用Jwts类的builder方法去创建这个JwtBuilder实例。我们要给这个实例设置三个属性：载荷、密钥、过期时间。然后JwtBuilder实例会自己生成 token 。

载荷实际是我们要存入 token 的信息，我们可以存一个用户 id ，在用户携带 token 的时候把这个 id 解析出来，再存到ThreadLocal里，方便参数传递，后续我会具体实现。此外，这个信息是以键值对的形式存储的。

密钥和过期时间都是自定义的，存到属性文件里面就行了。（过期时间 = 当前时间 + 有效时间）

我们可以写一个JwtProperties Bean 去读配置文件，然后把这个 Bean 交给IOC容器来管。

解析 token 是Jwt.parser()。这次只有两个参数，一个是要解析的 token ，一个是之前定义的密钥。最后解析出的结果是一个Claims不赘述，自行查看模板。

2. 本地线程变量

BaseContext.java

解析 token 会得到Claims，正是之前存入的用户信息，信息里面包含了用户 id 。我们可以把用户 id 解析出来后，存入ThreadLocal，方便参数的传递。

当然，设置 id 和 获取 id 的方法封装在BaseContext类里。

主要是它的使用，可以看看登录流程图，注意以下几点：

  1. 令牌给前端之后，每次请求带令牌是前端来做的，前端一般放在请求头里，也是 JSON 格式：'token' : '...'
  2. 后端要从请求头里拿令牌，那么应该什么时候拿呢？应该在除了/login和/register请求以外所有请求之前拿令牌然后解析。
  3. 因此我们需要自定义一个拦截器JwtTokenInterceptor，拦截除了/login和/register请求以外所有请求，并且解析令牌。
  4. 要让指定拦截哪些请求我们还需要在配置类WebConfig里注册拦截器，反正涉及到拦截请求的我们都要注册。
